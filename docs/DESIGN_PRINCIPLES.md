# claude-code-java 的设计原则

本文档解释了此工作区中 AI 技能和工作流的核心理念。

## 1. 人在回路中

- **开发者决策**：AI 提出更改；开发者审查并批准。
- **未经明确批准不得自动 commit 或写入 git**。
- 技能受控环境中生成输出，例如 diff 或只读视图。

## 2. 可复现性

- 技能是版本化的且可复用。
- 在相同上下文重新运行时，工作流应产生一致的结果。
- 指标和输出是可测量的：token 使用量、节省的时间、覆盖率改进。

## 3. 安全性

- AI 永远不应修改其定义范围之外的文件。
- 任何写入磁盘的技能应该：
  - 指示将更改哪些文件
  - 允许在应用前进行审查

## 4. 透明度

- 每个提议的更改都应该是可解释的：
  - 为什么需要更改
  - 哪些文件受影响
- 日志或解释应与 diff 一起提供。

## 5. 渐进式采用

- 从只读/分析技能开始。
- 转向提案技能。
- 只有在考虑带审查的变更技能时。
